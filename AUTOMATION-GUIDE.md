# Руководство по автоматизации курортного предприятия

> Применимо к: Курорт · Гостиницы · Рестораны · Магазин

---

## 1. Обратная связь от гостей (уже реализовано ✅)

Текущая система анкетирования через QR-код — отличная база. На её основе можно добавить:

- **Авто-уведомления** менеджеру при оценке 1–2 по любому критерию (email / Telegram-бот)
- **Еженедельный PDF-отчёт** со статистикой, отправляемый руководству автоматически
- **Дашборд в реальном времени** с графиками тrendов по каждому вопросу
- **Сравнение по корпусам / ресторанам** через room_id

## 2. Гостиница — фронт-офис

| Задача | Что автоматизировать | Инструмент |
|--------|----------------------|------------|
| Заселение / выселение | Онлайн-регистрация, автовыдача кода на дверь | PMS + IoT замки |
| Уборка | Расписание горничных по занятости номеров, статус «чисто/грязно» на планшете | Housekeeping-модуль |
| Техобслуживание | Заявки гостей через QR → задача исполнителю → push когда выполнено | Helpdesk-бот |
| Напоминания | За 24ч до выезда — SMS/email с предложением продлить | CRM + триггерные рассылки |
| Документооборот | Автозаполнение регистрационных карт из PMS | eSign / API ФМС |

## 3. Рестораны и бар

| Задача | Что автоматизировать | Эффект |
|--------|----------------------|--------|
| Приём заказов | QR-меню + самостоятельный заказ со стола | −20–30% нагрузка на официантов |
| Кухня | Экран заказов (KDS) вместо бумажных чеков | Скорость, меньше ошибок |
| Склад / списание | Автосписание ингредиентов при закрытии заказа | Точный остаток, нет пересорта |
| Бронирование столов | Онлайн-форма / Telegram-бот с подтверждением | 24/7 без участия персонала |
| Аллергены / КБЖУ | Автоматический расчёт при изменении меню | Соответствие требованиям |

## 4. Магазин (сувениры, продукты, аптека)

| Задача | Что автоматизировать | Эффект |
|--------|----------------------|--------|
| Касса | Облачная POS-система с эквайрингом | Скорость, меньше наличных |
| Остатки | Авто-заказ поставщику при достижении мин. остатка | Нет пересортицы |
| Ценники | Электронные ценники (ESL) — обновление с компьютера | Нет ручной замены |
| Аналитика | ABC-анализ продаж, топ товаров, мёртвые позиции | Управление ассортиментом |
| Лояльность | Электронные карты гостя (накопление баллов за покупки в магазине + ресторане + SPA) | Повторные покупки |

## 5. Курорт — общие сервисы

### SPA и анимация
- **Онлайн-запись** на процедуры / занятия (форма или бот) → автоматическое внесение в расписание
- **Напоминание** гостю за 1 час до процедуры
- **Загруженность** залов в реальном времени (датчики или ручной ввод)

### Безопасность и территория
- **СКУД** (контроль доступа) — единая карта гостя для бассейна, SPA, парковки
- **Видеонаблюдение** с аналитикой (подсчёт людей, очереди)
- **Мобильные обходы** — чек-листы на планшете вместо бумажных журналов

### Энергоэффективность
- **Умный термостат** — авто-выключение кондиционера при открытом окне или выезде гостя
- **Учёт энергопотребления** по корпусам → выявление аномалий

---

## 6. Единая CRM и программа лояльности

**Ключевая идея:** все точки контакта гостя (заезд, ресторан, магазин, SPA, анкета) → одна карточка гостя.

```
Гость → QR-регистрация → CRM-профиль
         ↓
 Ресторан / SPA / Магазин → баллы лояльности
         ↓
 Анкета при выезде → тег «доволен / не доволен»
         ↓
 Email через 30 дней: «Скучаем! Скидка 10% на следующий приезд»
```

**Метрики, которые сразу станут видны:**
- Средний чек в ресторане по типу гостя
- NPS (из анкеты) в разрезе источника бронирования
- Повторные заезды vs новые гости

## 7. Коммуникации с гостями

| Канал | Автоматизация |
|-------|--------------|
| WhatsApp / Telegram | Бот для заявок, бронирования, FAQ; передача живому оператору при сложных вопросах |
| Email | Цепочка: подтверждение → напоминание за 3 дня → приветствие в день заезда → «спасибо» при выезде |
| SMS | Код доступа к номеру, напоминание о процедуре, спецпредложение |
| Голос (IVR) | Автоответчик на стойке ресепшен в нерабочее время |

## 8. Аналитика и отчётность

Цель — **один дашборд** для директора, обновляемый в реальном времени:

- Загруженность гостиницы (%) сегодня / неделя / месяц
- Средняя оценка по анкетам (уже есть в API `/api/surveys/stats`)
- Выручка ресторана / магазина / SPA
- Топ жалоб из комментариев анкеты (частотный анализ текста)
- План-факт по KPI персонала

**Стек:** PostgreSQL (Supabase уже есть) → Metabase / Grafana / Power BI → автоотправка отчёта в Telegram директору каждое утро в 8:00.

## 9. Персонал

| Задача | Автоматизация |

| Связь | Корпоративный Telegram-канал с ботом: задачи, чеклисты, объявления |

## 10. Приоритеты внедрения (дорожная карта)

### Быстрые победы (1–3 месяца, минимум затрат)
1. **Telegram-бот** для уведомлений менеджера о плохих оценках (расширение текущей анкеты)
2. **QR-меню** в ресторане — любой конструктор меню
3. **Онлайн-бронирование** столов и SPA-процедур
4. **Автоотчёты** из существующей БД Supabase → Google Sheets / Metabase

### Средний приоритет (3–6 месяцев)
5. **Housekeeping-модуль** (мобильное приложение для горничных)
6. **Облачная касса** в магазине с учётом остатков
7. **Единая карта гостя** (программа лояльности)

### Стратегические проекты (6–18 месяцев)
8. **Полноценная PMS** для гостиницы с интеграциями
9. **CRM** с историей гостя и триггерными рассылками
10. **Умный учёт** энергии и СКУД

## Связь с текущим проектом

Существующая система анкетирования (`/api/surveys/*`) является **ядром** будущей аналитики. Следующие логичные шаги:

```
Текущее:  Анкета → Supabase → GET /stats
             ↓
Добавить: Авто-уведомление при оценке ≤ 2 → Telegram/Email менеджера
             ↓
Добавить: Еженедельный отчёт → PDF на почту
             ↓
Добавить: Дашборд → Metabase поверх Supabase (5 минут настройки)
```
